What is Open Source Software?

Open Source Software (OSS) is computer software whose source code is made publicly available for anyone to inspect, modify, and enhance.

Open Source Software
  • Source code is publicly available
  • Built on collaboration and sharing
  • Anyone can inspect, modify, enhance
  • Community-driven development
  • Examples: Linux, Firefox, WordPress
Proprietary Software
  • Source code is kept secret
  • Only the company can modify code
  • Users get compiled version only
  • Company-controlled development
  • Examples: Microsoft Office, Adobe Photoshop

The Four Essential Freedoms

The key rights granted by an open source license are the freedom to:

Freedom to Use

You are free to use the software for any purpose — personal, educational, or commercial.

Freedom to Study

You can study how the program works and understand its source code to learn from it.

Freedom to Modify

You have the right to change or improve the software to suit your own needs.

Freedom to Distribute

You may share copies — original or modified — so others can also benefit from the software.

Common Open Source Licenses

GPL (General Public License)
Copyleft

Requires any modified versions to also be open source. Ensures derivatives remain free.

LGPL (Lesser General Public License)
Weak Copyleft

Similar to GPL but allows linking with non-open-source software.

MIT License
Permissive

Very permissive; allows reuse with minimal restrictions. Simple and business-friendly.

Apache License
Permissive

Permissive and includes explicit patent protection for contributors.

Creative Commons (CC)
Content Focused

Often used for creative works like documentation, images, and media, not software code.

Key Concept: Copyleft vs Permissive

Copyleft licenses (like GPL) require that derivative works remain open source, creating a "viral" effect that preserves software freedom.

Permissive licenses (like MIT and Apache) allow more flexibility, including incorporation into proprietary software, making them popular for business use.

Benefits of Open Source

  • Transparency: Anyone can review the code for security and quality
  • Collaboration: Global community of developers can contribute improvements
  • Cost-Effective: No licensing fees for use
  • Customization: Users can modify software to meet specific needs
  • Innovation: Rapid development through collective effort
  • Longevity: Not dependent on a single company's survival
Critical Thinking Questions
  • Why might a company choose a permissive license over a copyleft license for their open source project?
  • What are the potential security advantages and disadvantages of open source software?
  • How does the open source development model differ from traditional proprietary software development?
  • What responsibilities do users have when they modify and redistribute open source software?
Research Assignment

Explore Real-World OSS Projects:

  • Choose three popular open source projects (e.g., Linux, Firefox, WordPress)
  • Identify which license each project uses and explain why that license might have been chosen
  • Research how each project manages community contributions and development
  • Write a brief report comparing their development models and community structures
